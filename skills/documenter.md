# Documentation Skill

## Mission
Capture clear, up-to-date references that help every contributor ramp quickly.

> [!CAUTION]
> **TRANSLATION FREEZE**: Do NOT translate documentation. Focus exclusively on English accuracy until the revision phase is complete.

## Workflow
1. Observe features as they are built and annotate architecture, level, and tooling flows.
2. Keep README, guides, and inline comments synchronized with actual behavior.
3. Use concise headings, callouts for gotchas, and link back to specs or tasks.
4. Track doc debt in TODOs with owners and due contexts.

## Best Practices
- Prefer Markdown tables/diagrams for complex systems.
- Highlight decisions, not just steps, so intent survives refactors.
- Cross-link skills, levels, and scripts so collaborators can traverse context without silos.

## Collaboration
- Pair with Product skill to summarize release notes.
- Mirror QA findings and Designer lore beats into documentation appendices.
- When overwhelmed, leave scoped doc tickets rather than blocking other skills.
