Here’s an updated approach to **apply the triangle + CRT shader only to the game world while keeping UI fully unaffected**. This uses **Viewports** and **ViewportTexture** so the shader never touches UI elements.

---

## 1. **Scene Structure**

```
ShaderMaskDemo (Node2D)
 ├── ViewportContainer (fills screen)
 │	└── Viewport (size = window)
 │		 ├── World (Node2D)
 │		 │	├── Sprite1 (background/game elements)
 │		 │	└── Sprite2 (moving objects)
 │		 └── ColorRect (FullRect, ShaderMaterial applied)
 └── CanvasLayer (Layer 2) → UI above filter
	  └── VBoxContainer
		   ├── HSlider (triangle_size)
		   ├── HSlider (edge_softness)
		   ├── HSlider (scanline_intensity)
		   ├── ColorPickerButton (color1)
		   └── Button (Toggle Filter)
```

**Key idea:**

* **World Node** contains all game visuals.
* The **ColorRect shader** samples the `ViewportTexture` from the Viewport.
* UI is outside the viewport and renders above the filter.

---

## 2. **Shader Code** (`triangle_crt_viewport.shader`)

```gdshader
shader_type canvas_item;

uniform sampler2D world_texture : hint_albedo; // assign ViewportTexture

// Filter toggle
uniform bool enable_filter : hint_range(0,1) = true;

// Triangle settings
uniform float triangle_size : hint_range(1.0, 50.0) = 10.0;
uniform vec3 color1 : hint_color = vec3(1.0, 1.0, 1.0);
uniform vec3 color2 : hint_color = vec3(0.0, 0.0, 0.0);

// CRT/softening settings
uniform float scanline_intensity : hint_range(0.0, 1.0) = 0.15;
uniform float edge_softness : hint_range(0.0, 0.5) = 0.05;

vec3 trianglePattern(vec2 uv) {
	float row = floor(uv.y);
	float col = floor(uv.x - mod(row, 2.0) * 0.5);
	vec2 local = vec2(uv.x - col - mod(row, 2.0) * 0.5, uv.y - row);
	float tri = smoothstep(0.0, edge_softness, min(local.x, 1.0 - local.x) - local.y);
	float checker = mod(row + col, 2.0);
	return mix(color1, color2, checker * tri);
}

void fragment() {
	vec4 scene_color = texture(world_texture, SCREEN_UV);

	if (enable_filter) {
		vec2 uv = FRAGCOORD.xy / triangle_size;
		vec3 tri_color = trianglePattern(uv);
		vec3 color = mix(scene_color.rgb, tri_color, 0.7);

		float scan = sin(FRAGCOORD.y * 3.1415 / 2.0) * scanline_intensity;
		color *= 1.0 - scan;

		vec2 norm = SCREEN_UV * 2.0 - 1.0;
		float vign = smoothstep(1.0, 0.7, length(norm));
		color *= vign;

		COLOR = vec4(color, 1.0);
	} else {
		COLOR = scene_color;
	}
}
```

## 3. **GDScript for UI Controls** (`ShaderMaskDemo.gd`)

```gdscript
extends Node2D

@onready var color_rect := $ViewportContainer/Viewport/ColorRect
@onready var shader_mat := color_rect.material as ShaderMaterial
@onready var toggle_button := $CanvasLayer/VBoxContainer/ToggleButton

func _ready():
	# Assign ViewportTexture
	var vp_tex = $ViewportContainer/Viewport.get_texture()
	shader_mat.set_shader_parameter("world_texture", vp_tex)

	# Connect sliders
	$CanvasLayer/VBoxContainer/HSlider_triangle_size.connect("value_changed", self, "_on_triangle_size_changed")
	$CanvasLayer/VBoxContainer/HSlider_edge_softness.connect("value_changed", self, "_on_edge_softness_changed")
	$CanvasLayer/VBoxContainer/HSlider_scanline_intensity.connect("value_changed", self, "_on_scanline_intensity_changed")
	$CanvasLayer/VBoxContainer/ColorPickerButton_color1.connect("color_changed", self, "_on_color1_changed")

	# Connect toggle button
	toggle_button.connect("pressed", self, "_on_toggle_filter_pressed")

func _on_triangle_size_changed(value):
	shader_mat.set_shader_parameter("triangle_size", value)

func _on_edge_softness_changed(value):
	shader_mat.set_shader_parameter("edge_softness", value)

func _on_scanline_intensity_changed(value):
	shader_mat.set_shader_parameter("scanline_intensity", value)

func _on_color1_changed(color):
	shader_mat.set_shader_parameter("color1", color.to_rgb())

func _on_toggle_filter_pressed():
	var current = shader_mat.get_shader_parameter("enable_filter")
	shader_mat.set_shader_parameter("enable_filter", not current)
```

---

## 4. **Key Advantages**

1. **UI unaffected:** Everything in `CanvasLayer` is above the viewport shader.
2. **Toggle filter:** Use the button to enable/disable the effect dynamically.
3. **Real-time adjustment:** Sliders modify triangle size, softness, scanlines, and colors.
4. **Safe scaling:** Shader always uses `ViewportTexture`, so resolution changes don’t break tiling.
